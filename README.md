**Parking Information System — Database & System Design**

Design and implementation of an information system for managing a parking operator’s workflow, including full system analysis, business rules modeling and relational database design.

**Project Overview**

The goal of the project was to design a real-world information system starting from business requirements and ending with a working database prototype.
The system supports multiple user roles and manages parking sessions, reservations, billing and maintenance processes.

**Responsibilities & Work Done**

- Performed business analysis: actors, goals, workflows and lifecycle events
- Designed UML diagrams (use-case, activity and system structure)
- Created conceptual and logical ER data model
- Defined business rules and integrity constraints (PK, FK, validation)
- Designed CRUD operation matrix and data contracts
- Specified functional and non-functional requirements (security, recovery, availability)
- Modeled role-based access control (admin, manager, client, analyst)
- Implemented working prototype in Microsoft Access

**Technologies & Methods**

- Relational Database Design
- SQL
- UML Modeling
- Requirements Engineering
- Microsoft Access (prototype implementation)
